# B&B CRM

**Bridges and Blueprints CRM** — a modern, minimal customer relationship manager for tracking companies, people, pipeline, and follow-ups.

Built with Next.js 16, TypeScript, Tailwind CSS 4, Prisma (SQLite), and Electron for desktop packaging.

---

## Features

- **Dashboard** — Pipeline stats, follow-up today / this week / overdue, total / new / in-progress leads, 14-day activity chart
- **Companies** — Full pipeline tracking (Lead Source, Lead Status, Pipeline Stage, Priority, Won/Lost/Open, Estimated Deal, ARR, Loss Reason), configurable columns, ideal profile flag, multiple contacts
- **People** — Contact profiles linked to companies, with notes timeline
- **Notes** — Multiple notes per company or person, with edit/delete
- **Settings** — User profile (drives "Created By" column), full CRUD on every dropdown option with colors, configurable Companies table columns
- **Themes** — Light, Dark, System (auto-persisted)
- **Keyboard shortcuts** — `⌘/Ctrl+K` command palette, `G then D/C/P/N/S` navigation, `T` theme toggle, `⌘/Ctrl+[` back, `⌘/Ctrl+]` forward — Mac and Windows aware
- **Cross-platform desktop apps** — Windows `.exe`, macOS `.dmg` (Intel + Apple Silicon), Linux `.AppImage`

## Build desktop installers

Installers are built in the cloud via GitHub Actions — no local build required.

See **[DESKTOP_SETUP.md](./DESKTOP_SETUP.md)** for the full step-by-step guide.

**TL;DR:**
1. Go to the **Actions** tab → **Build Desktop Installers** → **Run workflow**
2. Enter version `1.0.0`
3. Wait ~6 minutes
4. Download `B&B-CRM-Setup-1.0.0.exe` and `B&B CRM-1.0.0.dmg` from the run page

## Local development

```bash
# Install dependencies
bun install

# Push database schema + generate Prisma client
bun run db:push

# Seed demo data (30 companies, ~80 people, ~80 notes, dropdown options, user profile)
bun run seed

# Start dev server
bun run dev
```

Then open http://localhost:3000.

## Tech stack

| Layer | Tech |
|---|---|
| Framework | Next.js 16 (App Router) + TypeScript 5 |
| Styling | Tailwind CSS 4 + shadcn/ui (New York) |
| Database | Prisma ORM + SQLite |
| State | Zustand (nav) + custom `useFetch` hook |
| Charts | Recharts |
| Desktop | Electron 42 + electron-builder 26 |
| Themes | next-themes |
| Notifications | Sonner |

## Project structure

```
.
├── electron/                # Electron main process + preload
│   ├── main.ts              # Source — boots Next.js, opens window, native menus
│   ├── main.cjs             # Compiled (generated by `bun run build:electron`)
│   └── preload.cjs          # Secure IPC bridge to renderer
├── .github/workflows/
│   └── build-desktop.yml    # CI: builds Win + Mac + Linux installers
├── prisma/
│   └── schema.prisma        # Database models
├── scripts/
│   ├── seed.ts              # Demo data seeder
│   ├── generate-icons.ts    # PNG → .ico / .icns
│   ├── build-electron.ts    # Compile main + copy Next.js standalone
│   └── push-to-github.sh    # One-shot script to push this code to GitHub
├── src/
│   ├── app/                 # Next.js routes + API routes
│   ├── components/
│   │   ├── ui/              # shadcn/ui primitives
│   │   └── crm/             # B&B CRM components + views
│   ├── hooks/               # useFetch, useDropdownOptions, etc.
│   ├── lib/                 # db, types, utils, format
│   └── stores/              # Zustand nav store
├── electron-builder.yml     # Desktop build config (Win/Mac/Linux targets)
├── DESKTOP_SETUP.md         # Step-by-step desktop build guide
└── package.json
```
